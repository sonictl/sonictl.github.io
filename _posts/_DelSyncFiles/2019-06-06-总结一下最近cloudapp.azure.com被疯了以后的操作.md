### 总结一下最近cloudapp.azure.com被疯了以后的操作

本文日期：20190606

------



1. 用Google Cloud Service的VPS

   - 其中ssh key 的生成不一定要用 puttyGen 这个工具，直接secureCRT里就有这个功能。

2. 用ssh tunnel方式实现慢速电脑端上网

   - Mac上也有secureCRT的mac版。

   - 不用传统的myTunnel(不支持key方式免密登录)，

   - 而在secureCRT的'session Option>> Port forwarding -> local port forwarding '处添加一个，

       - 名称，
       - [\*] manually IP, 
       - IP=127.0.0.1, port=7070 ,
       - [\*]use socks 4 or 5

   - 就可以生成ssh tunnel了。还可以勾选’不请求shell‘

       

3.  配置wireGuard VPN 实现高速上网，Android, windows, Mac 皆可。

   1. 服务端的安装及配置。

      - 主要参考：

        - https://doubibackup.com/qbc20cn3.html
        - <https://blog.myhro.info/2018/09/quick-and-easy-vpns-with-wireguard>
        - wireGuard官网

      - 客户端client.conf在配置时注意 ip:port 参数，有点坑。

        - ip，得是vps的公网ip

        - 端口，需要先用命令sudo wg看服务端运行状态，从而得知端口

          

   2. Windows 客户端

      - win / android 都是参考的<https://doubibackup.com/t86-fo_1.html>

      - Windows 涉及到安装虚拟网卡，自动可能会失败，改用’TunSafe‘包中的文件：

        - 自 <https://tunsafe.com/download> 下载。
        - TunSafe X.X Standalone Zip for 64-bit systems (x64, Windows 7 or later, 206 kB))
        - TAP 虚拟网卡：Standalone TunSafe-TAP Installer 9.21.2 (Windows 7 or later, 178 kB)

      - 国内外分流的配置：

        - 这个VPN虽然速度够快，但还是没有 ssh Tunnel 或 ss那样在浏览器上switch Omega 切换那么方便。sshTunnel 基本够用。
- 下载auto-add-routes[批处理文件](https://github.com/lmc999/auto-add-routes/archive/master.zip)，修改TunSafe的client.conf文件，在[interface]部分（[peer]行上方）加入以下2行，并勾选 **Options - Allow Pre/Post Commands** 选项（不要在连接/断开期间动这个选项，清路由表故障）。
          PreUp = start "" "F:\crack_GFW_2019\TunSafe\auto-add-routes-master\wireguard\routes-up.bat"
          PostDown = start "" "F:\crack_GFW_2019\TunSafe\auto-add-routes-master\wireguard\routes-down.bat"
        - 参考： <https://doubibackup.com/x2_8j160.html>
- TunSafe.exe及其所在文件夹挪动位置后可能会报错找不到client.conf文件，disconnect一下再试。
        - 分流时遇到 “启动cmroute.dll时出现问题” 报错。要修复这个问题又是个坑了。暂时放下。
