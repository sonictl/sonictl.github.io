# MacOS 局域网屏幕共享方案

【场景描述】：要将一台 MacBook 的屏幕内容投屏到局域网内的另一台电脑上（linux or windows），使之拥有一个副屏显示器。

有两种方案。

# 方案1： UxPlay 运行在副屏电脑上，在局域网内启动一个AirPlay服务

如题，在从机上安装uxplay, 可以启动一个AirPlay服务供 Mac 电脑发现后直接投屏。

经Ubuntu 25.10系统上验证，uxplay工作稳定免费。

**可做首选方案～**



# 方案2：macOS + simpleDisplay + Deskreen 局域网虚拟屏扩展：屏幕共享方案

为了实现跨设备副屏效果，将另一台电脑作为主电脑的副屏显示器，本方案基于**simpleDisplay**开源虚拟显示器工具与**Deskreen**屏幕共享工具组合，在 macOS 设备上快速创建虚拟显示屏。通过局域网将主屏、虚拟屏或指定应用窗口画面，实时共享至同一网络内其他终端。接收设备无需安装专用客户端，仅通过浏览器输入地址即可访问并使用扩展屏幕，实现跨设备副屏效果，整套工具开源免费、部署简便、运行稳定。

## 

1. **simpleDisplay**：开源轻量虚拟显示器软件，原生适配 macOS，可自主创建自定义分辨率虚拟屏幕，驱动纯净，无多余后台残留，不影响系统显示与光标正常使用。
2. **Deskreen**：跨平台开源屏幕共享工具，依托 WebRTC 协议传输画面，支持整屏、虚拟屏、独立窗口三种共享模式，局域网内传输延迟低，兼容各类主流浏览器。

## 

1. 主控端：macOS 系统，接入局域网（有线 / 无线网络均可）。
2. 接收端：任意 PC、Linux 设备、移动设备，搭载主流浏览器，与主控端处于同一局域网。

## 四、部署与使用步骤

### （一）主控端软件安装

1. 安装 simpleDisplay

   访问官网 [https://simpleDisplay.app](https://link.wtturl.cn/?target=https%3A%2F%2FsimpleDisplay.app&scene=im&aid=497858&lang=zh) 下载 macOS 版本，将软件移入应用程序目录。首次运行时，在系统**安全性与隐私**中完成开发者授权，并开启辅助功能权限。

2. 安装 Deskreen

   前往 Deskreen 官方网站下载对应 macOS 安装包，完成安装后，在系统**安全性与隐私 - 屏幕录制**中为软件授予录屏权限，保障画面正常采集。

### （二）创建虚拟显示屏

1. 启动 simpleDisplay，点击界面添加按钮，按需设置虚拟屏分辨率、刷新率等参数，点击创建生成虚拟显示器。
2. 进入 macOS**系统设置 - 显示器**，在排列界面中将虚拟屏设置为扩展模式，按需调整屏幕相对位置。

### （三）配置屏幕共享

1. 运行 Deskreen，软件自动读取本机局域网 IP 与访问端口。
2. 根据使用需求选择共享类型：整机主屏、已创建的虚拟显示屏、单个应用窗口。
3. 确认配置后启动共享，软件生成对应的访问链接与二维码，保持程序后台运行。

### （四）接收端接入使用

1. 接收设备连接至同一局域网，打开任意浏览器。
2. 在地址栏输入主控端 IP + 端口，或直接扫描二维码，发起连接请求。
3. 主控端通过连接授权后，接收端浏览器即可加载画面，实现副屏扩展、窗口拖拽与内容查看。

## 常见问题处理

1. **无法创建虚拟屏**

   检查软件权限是否完整授予，重启 simpleDisplay 后重新尝试。

2. **接收端无法访问**

   关闭 macOS 系统防火墙，或在防火墙规则中放行对应访问端口；确认两端设备 IP 处于同一网段。

3. **画面传输卡顿**

   优先使用有线网络或 5G 频段 WiFi，适当降低共享画面分辨率，关闭主控端高负载后台程序。

## 方案特点

1. **开源免费**：全部组件无付费项、无功能阉割，使用成本低。
2. **稳定性强**：虚拟屏驱动简洁，不会出现光标丢失、桌面异常等问题。
3. **使用灵活**：支持整屏、虚拟屏、独立窗口多种共享模式，适配不同使用场景。
4. **跨平台兼容**：接收端仅依赖浏览器，支持 Linux、Windows、移动设备等各类终端。
